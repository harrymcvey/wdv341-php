1. version control software - keeps track of every modification to the code in a special kind of database 2. add - adds a change in the working directory to the staging area 3. commit - commit a snapshot of all changes in the working directory 4. push - used to upload local repository content to a remote repository 5. pull - used to fetch and download content from a remote repository and immediately update the local repository to match that content 6. clone -primarily used to point to an existing repo and make a clone or copy of that repo at in a new directory, at another location
